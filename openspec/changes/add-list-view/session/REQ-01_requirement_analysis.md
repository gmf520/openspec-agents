# REQ-01: 需求分析 — add-list-view

## 背景

当前 todos 应用仅支持看板视图（Board.vue），卡片按列分组横向展示，支持拖拽移动。用户希望增加一个列表视图，与看板可互相切换，提供不同的信息浏览方式。

## 用户故事

- **作为** 任务管理者
- **我想要** 以纵向列表方式查看所有任务卡片
- **以便** 在任务较多时能更高效地浏览和定位卡片

## 功能需求

| ID | 需求 | 优先级 |
|----|------|--------|
| R1 | 工具栏提供视图切换按钮（看板 / 列表），互相切换 | P0 |
| R2 | 列表视图以卡片式纵向排列，按列（状态）分组 | P0 |
| R3 | 列表卡片复用看板卡片的全部信息展示（标题、描述、标签、优先级、截止日期、完成状态） | P0 |
| R4 | 列表视图支持 checkbox 勾选完成/取消完成 | P0 |
| R5 | 列表视图仅查看模式，点击编辑/删除按钮弹出 CardModal | P0 |
| R6 | 列表视图共享搜索和筛选状态 | P0 |
| R7 | 视图偏好存入 localStorage，刷新后恢复 | P0 |
| R8 | 空列分组不渲染，所有列无匹配时显示"未找到匹配的卡片" | P1 |
| R9 | 分组标题可折叠（默认展开） | P2 |

## 非功能需求

- 列表卡片不带拖拽功能（`cursor: default`）
- 与 Board 视图互斥渲染，不重复挂载
- 无新增依赖

## 确认项

- [x] 卡片式列表（非紧凑表格）
- [x] 仅查看模式（点击弹出编辑弹窗）
- [x] 不需要排序
- [x] 记住视图偏好
